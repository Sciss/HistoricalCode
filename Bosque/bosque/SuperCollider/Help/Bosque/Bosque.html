Bosque


Bosque is an application built on top of SuperCollider and SwingOSC. It is a successor of the software used for the "Forest 2" project during the Weimar TanzMedienAkademie 2007 ("bosque" = Spanish for forest). Version 0.29 is the result of a visit at the IEM Graz in July 2009 for the EGM (embodied generative music) project.


Bosque is basically a multi-track editor that allows you to place regions in a timeline and playback that timeline. Regions can be audio files, breakpoint functions (envelopes) or SuperCollider code fragments. The timeline and transport can be programmatically controlled via SuperCollider. The editor's display window is a custom Java component accessed via the SwingOSC framework.


The following help files are available:


Bosque_Installation - which files are needed and where they go

Bosque_Configuration - how to set up and launch the application

Bosque_Editing - basic tutorial for the timeline editor

Bosque_Audio - description of the audio engine



Bosque Class


*launch


Instantiates and launches the Bosque application. If no scsynth server has been booted, it will do this as well. The new Bosque instance is available through the *default method.


Bosque.launch;


*default


Returns the default instance of Bosque. This class field is automatically set when Bosque is instantiated for the first time (using *new or *launch).


*version


Returns the version (Float) of the Bosque class library.


*soundCard, *numInputBusChannels, *numOutputBusChannels, *workDir, *masterBusNumChannels, *masterChanMap, *timeBasedJar, *midiInDev, *midiOutDev


These are covered in the Bosque_Configuration help file.


*track( <(Symbol) name> )


Shortcut for returning a BosqueTrack from the current session.


*mark( <(Symbol) name> )


Shortcut for returning a BosqueMarkerStake from the current session.


swing


Returns the SwingOSC server used for the Bosque GUI.


scsynth


Returns the scsynth Server used for the Bosque audio engine.


session


Returns the current BosqueSession document. Bosque is a single-document-application (SDA), so there is always exactly one document available.


masterGroup, masterVolume, diskGroup, preFilterGroup, postFilterGroup, panGroup, mixGroup, masterBus


These are covered in the Bosque_Audio help file.


timelineEditor


returns an instance of BosqueTimelineEditor, the class resposible for setting up the main Bosque GUI.
