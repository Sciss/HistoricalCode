BosqueSessionCollection


The BosqueSessionCollection is a class with many similarities to a regular SuperCollider SequenceableCollection class: If has methods to add or remove elements, and those elementsÂ  can be accessed by an Integer index.


However, BosqueSessionCollection is specialized in that it assumes its elements implement some methods, e.g. name to return their names, and its manipulation methods usually take a source object as their first argument, indicating that this source is the originator of the manipulation. Whenever source is not nil, the collection will notify registered observers about the manipulation.


add( <(Object) source>, <(Object) object> )


Adds an object to the collection. If source is not nil, delivers a notification ( \add, object ) to registered observers. Note that object.name must be unique, e.g. there cannot be two objects in the collection that have the same name. If you try to add an object with a name that is already found in the collection, an error is thrown.



addAll( <(Object) source>, <(Collection) objects> )


Adds several objects to the collection. If source is not nil, delivers a notification ( \add, *objects ) to registered observers.



remove( <(Object) source>, <(Object) object> )


Removes an object from the collection. If source is not nil and the object was included in the collection, delivers a notification ( \remove, object ) to registered observers.



removeAll( <(Object) source>, <(Collection) objects> )


Removes several objects from the collection. If source is not nil, delivers a notification ( \remove, *objects ) to registered observers.



size


Returns the number of elements in the collection (Integer).



isEmpty, notEmpty


Returns whether the collection is empty or not empty (Boolean).



at( <(Integer) index> )


Returns the element at a given index, or nil if the index is out of range.



indexOf


Returns the Integer index of an element in the collection, or nil if the element is not included in the collection.



getAll


Returns a SequenceableCollection with all elements in the collection. This is a copy so it's safe to manipulate the result.



do, collect, select, detect


Convenience methods that reproduce some of the regular SequenceableCollection behaviour.



find( <(Symbol) name> )


Returns the element in the collection that has a given name, or nil if no element with that name was found.



includes( <(Object) object> )


Detects whether an object is included in the collection or not (Boolean).



createUniqueName( <(String) ptrn>, <(Integer) count> )


Creates a unique name for an element to be inserted into the collection. This method takes a ptrn (pattern) argument and a start count (counter) argument to synthesize the name. The pattern string should contain a percent character % which is replaced by the count argument. If a name with a given count already exists in the collection, the count value is incremented and a new name is synthesized, until a unique name has been found and is returned (Symbol). Example:


myCollection.createUniqueName( "Track_%", 1 );
