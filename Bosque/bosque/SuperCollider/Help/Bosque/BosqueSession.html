BosqueSession


This class represents the document which contains all the tracks and regions, timeline information etc. The session can be loaded and saved via the BosqueTimelineEditor. Since Bosque is a single-document-application, the session can always be accessed via


~doc = Bosque.default.session;


Instance fields / getter methods


timeline


Returns the BosqueTimeline instance.


timelineView


Returns the BosqueTimelineView instance which allows you to access the BosqueTimelineCursor and the BosqueTimelineSelection.


transport


Returns the BosqueTransport instance.


tracks


Returns a BosqueSessionCollection containing all BosqueTrack objects.


selectedTracks


Returns a BosqueSessionCollection containing all currently selected BosqueTrack objects.


audioFiles


Returns a BosqueSessionCollection containing all BosqueAudioFile objects.


busConfigs


Returns a BosqueSessionCollection containing all BosqueBusConfig objects.


trail


Returns a BosqueTrail containing all BosqueRegionStake objects.


selectedRegions


Returns a BosqueSessionCollection containing all currently selected BosqueRegionStake objects.


audioPlayer


Returns the BosqueAudioPlayer instance.


markers


Returns a BosqueTrail containing all BosqueMarkerStake objects.


path


Returns the file (String) to which the session was saved, or nil.


name


Returns the name of the session, i.e. a human readable String take from the path.


undoManager


Returns the ScissUndoManager instance.


isDirty


Returns a Boolean indicating whether unsaved modifications have been done to the session.



Instance methods


editPosition( <(Object) source>, <(Integer) pos> )


Sets the timeline view's cursor position to a new value. source refers to the object originating the modification. Example:


~doc.editPosition( -1, 22050 ); Â  // go to 0:00.500



editSelect( <(Object) source>, <(Span) span> )


Sets the timeline view's selection Span to a new value. source refers to the object originating the modification. Example:


~doc.editSelect( -1, Span( 10000, 100000 ));


editScroll( <(Object) source>, <(Span) span> )


Sets the timeline view's visible Span to a new value. source refers to the object originating the modification. Example:


~doc.editScroll( -1, Span( 0, 88200 )); // zoom to seconds 0...2


editInsertTimeSpan( <(Object) source>, <(Integer) span>, <(JSyncCompoundEdit) ce> )


Adds the operation of inserting a time span into the session to a given compound edit object. You will most likely use the more simple way of calling BosqueTimelineEditor's insertSpan method instead.



editRemoveTimeSpan( <(Object) source>, <(Integer) span>, <(JSyncCompoundEdit) ce> )


Adds the operation of removing a time span from the session to a given compound edit object. You will most likely use the more simple way of calling BosqueTimelineEditor's removeSpan method instead.



editClearTimeSpan( <(Object) source>, <(Integer) span>, <(JSyncCompoundEdit) ce>, <(Function) filter> )


Adds the operation of clearing a time span in the session to a given compound edit object. You will most likely use the more simple way of calling BosqueTimelineEditor's clearSpan method instead.



clear


Clears the session, that is it removes all its objects and resets the timeline. Warning: there is no way to undo this operation and this method will not ask you to confirm when there are unsaved changes to the document.



Kontur


Rudimentary export of Bosque sessions to Kontur is provided:


w = File( "~/Desktop/TestSession".absolutePath, "w" );

w.put( Bosque.default.session.konturExport );

w.close;
