// ----- prerequisites -----

// a SwingOSC server (v0.60) needs to be running

SwingOSC.default.boot;


// !!! SuperCollider: OSCpathResponder is partly broken in SC 3.2,

// hence a recent SVN version of SuperCollider must be used !!!


// ----- define the hardware setup -----

// you see, these are not saved with the sessions

(

Bosque.soundCard = "Built-in Audio";

Bosque.numInputBusChannels = 8;

Bosque.numOutputBusChannels = 8;

Bosque.workDir = "~/Bosque/".standardizePath; // ! waveform cache goes here !

Bosque.masterBusNumChannels = 8; // the size of the master bus

Bosque.masterChanMap = [ 0, 1, 2, 3, 4, 5, 6, 7 ];  // define the master channel indices

// location of the timebased library

Bosque.timeBasedJar = "file:///Users/rutz/Documents/workspace/TimeBased/TimeBased.jar";

Bosque.createSynthDefs;

)


// ----- launch the app -----

Bosque.launch;


// ----- creating a basic session in the GUI -----

// creating some time span

• Timeline Window -> Timeline Menu -> Insert Span...

// creating some tracks

• Menu Window -> Collections - > Tracks Tab -> + (do this e.g. four times)

// creating a mono-to-stereo and stereo-to-stereo output bus

• Menu Window -> Collections - > Busses Tab -> +

• name this bus "Mono2Stereo", it shall have 1 input channel and 2 output channels

• in the grid view, route the input to both outputs, drag down the gain slider

  so they go out at -3 dB ; confirm by hitting the "Apply" button

 

! Note: bus indices are logical, e.g. index 0 corresponds to the first

  master bus channel. Which hardware channel this is depends on masterChanMap !

 

• Menu Window -> Collections - > Busses Tab -> +

• name this bus "Stereo2Stereo", it shall have 2 input and 2 output channels

• in the grid view, route the first input to the first output, and the

  second input to the second output (a diagonal matrix)

  ; confirm by hitting the "Apply" button

// import some audio files

• Menu Window -> Collections - > Audio Files Tab -> +

• select a mono audio file

• Menu Window -> Collections - > Audio Files Tab -> +

• select a stereo audio file

// place the audio files in the timeline

• In the Timeline Window select the first track by clicking on its name ("Track 1")

• Place the timeline position pointer on the insertion spot by pressing the mouse

  button over the timeline axis view (top part of the window) and moving the

  mouse while keeping the button pressed

• In the Audio Files Tab of the Collections Window, select the mono audio file

  and choose "Drop Region" from the Action menu.

• Open the Observer window by clicking on the "Observer" button in the Menu Window,

  select the "Track" tab : we will need to assign a bus configuration to this track.

• Simultaneously open the Collections window and go to the "Busses" tab. Keep the

  Control key pressed and press the mouse button over the Mono2Stereo entry. Keep

  The mouse button pressed and move the move to the "Bus Config" drag-sink view in

  the observer`s track tab. The drag-sink should now read "Mono2Stereo"

• Repeat the same for the stereo audio file and "Track 2". Assign the "Stereo2Stereo"

  bus config to this second track in the same manner.

// playback the result

• Locate the Transport Window and press the play button. You should hear the two

  tracks playing back the audio files on the defined busses.

  

// ----- doing basic editing -----

// splice a region

• Select the region to splice in the timeline editor by pressing the mouse over the

  region (the region becomes colour-inverted).

• Place the timeline position (by dragging in the timeline axis) at the splice point

• Choose Timeline -> Splice Objects from the timeline editor`s menu bar

• To create an overlap, drag the second splice to the next track and move it back

  in time. Don`t forget to assign an appropriate bus config to this third track,

  unless you have done do before.

// shortening a region

• To shorten the newly created splice, switch from the "Move" tool to the "Resize"

  tool in the toolbar of the timeline editor window. To truncate the region`s beginning,

  locate the mouse near the beginning of region, to truncate the ending, locate the mouse

  near the ending of region. Press the mouse button and move the mouse to adjust the

  region bounds.

// applying a fadein and gain change

• Select the region to customize

• Select the "Regions" tab in the observer window

• Enter a fade-in or fade-out time in milliseconds; confirm changes by hitting return!

• Enter a gain-correction in decibels; note how the waveform view reflects the new gain

// renaming and colourizing a region

• In the same "Regions" tab, you can alter the name and the colour of the region

// cutting out a portion of the timeline

• Alt+click in the timeline axis to clear the timeline selection and to the define

  the selection start. Shift+click in the axis to make the timeline selection.

• Choose Timeline -> Remove Span from the timeline editors menu bar. Try Edit -> Undo / Redo

  if you like.


// ----- storing the session -----

• Don`t forget to save the session from time to time: In the timeline editor window,

  choose File -> Save As... from the menu. The resulting file is in plain text

  format, so you could open it in a text editor and do modfications (e.g. change

  the folder where the soundfiles are to be found)
