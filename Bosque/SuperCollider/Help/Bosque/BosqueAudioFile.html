BosqueAudioFile


This class represents an audio file contained in a BosqueSession. A BosqueSessionCollection containing all BosqueAudioFile objects is found via


~doc = Bosque.default.session;

~audioFiles = ~doc.audioFiles;


*<>overviews


This global Boolean parameter specifies whether the BosqueTimelineEditor window should display BosqueAudioRegionStake objects with a corresponding waveform overview or not. The current implementation is very bad regarding memory usage (since for each audio file a separate JSCSoundFileView with its own dedicated double buffered image is kept), so you might encounter OutOfMemoryErrors when using two many audio files with overviews enabled. In that case, the only solution at the moment is to switch of overviews (using BosueqAudioFile.overviews_( false ) before launching Bosque).



path


Returns the full path (String) of the audio file. Currently, no information about relative directory structures is stored with a BosqueSession, so when you copy a session along with its audio files to a new location, you will get errors when trying to open the session. The current workaround is to open the session .txt file with a text editor and using find-and-replace to move the audio file paths to the new location.



name


The human readable name (String) of the audio file, basically the filename without extension.



numChannels


Returns the number of channels (Integer) the audio file has.



numFrames


Returns the number of sample frames (Integer) the audio file has.
