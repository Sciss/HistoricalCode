Bosque - Installation


Bosque is currently distributed in the form of Subversion (SVN) repository as part of my Ph.D. project whose worktitle is Tintantmare : http://sourceforge.net/projects/tintantmare/ .


In the current version, there are two related branches which must be installed. In a future version, these branches might be fusioned together. The branches are bosque (the SuperCollider classes) and timebased (the Java classes). To create a new checkout:


$ cd <mainDir>

$ svn co https://tintantmare.svn.sourceforge.net/svnroot/tintantmare/bosque bosque

$ svn co https://tintantmare.svn.sourceforge.net/svnroot/tintantmare/timebased timebased


The SuperCollider classes and help files from the bosque folder should be installed in a proper extensions location. The best approach is to create a symlink so updating the working copy is automatically reflected in SuperCollider:


$ ln -s <mainDir>/bosque/SuperCollider <extDir/bosque


where <extDir> typically is


thisProcess.platform.userExtensionDir


Bosque relies on some classes from my private library, e.g. the timebased framework that was independantly developed, that is why there are three class files folders (bosque, sciss, egm2).


The Java extensions file TimeBased.jar can basically remain in your working copy location since you will set up the path in the Bosque_Configuration process.


SuperCollider and SwingOSC versions


- Typically a recent SVN version of SuperCollider 3.2+ should be used. I am successfully using revision 7716.

- While a suitable SwingOSC.jar binary is included in the timebased branch, any current SVN version of SwingOSC should work. I am successfully using revision 95.
