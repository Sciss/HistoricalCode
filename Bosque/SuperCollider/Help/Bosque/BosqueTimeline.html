BosqueTimeline


The timeline object represents the complete time span covered by a BosqueSession. It is accessible via


~timeline = Bosque.default.session.timeline;


span


Returns the Span that the timeline covers, usually starting at frame 0. Units are frames referring to the samplerate given by the rate field of the timeline. For example, to determine the duration of the session in seconds:


(~timeline.span.length / ~timeline.rate).asTimeString;


When the timeline's span is modified using the span_ setter method (you are advised to do this indirectly, e.g. using the editInsertSpan or editRemoveSpan methods of BosqueTimelineEditor), a notification ( \span, newSpan ) is delivered to registered observers.



rate


Returns the samplerate or time base used to express frame positions in the document. By default, a rate of 44100 is used. Note that this doesn't imply that the sound files used in the session need to have a samplerate of 44.1 kHz, nor does it mean that the scsynth server must run at 44.1 kHz. It merely defines the resolution in which timeline positions can be expressed.



rate_( <(Float) newValue> )


Sets the timeline's time base to the given value in frames per second. There is currently no undo manager aware method to modify the rate. It is suggested that you adjust the session's rate immediately after clearing or opening the session in order not to conflict with the undo manager.


When the rate is adjusted, a notification ( \rate, newValue ) is delivered to registered observers.
