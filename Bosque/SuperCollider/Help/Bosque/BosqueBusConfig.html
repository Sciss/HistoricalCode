BosqueBusConfig


A bus configuration describes the mapping of a set of input audio channels to a set of output audio channels. Each BosqueTrack can be associated with a BosqueBusConfig. In order to make the BosqueAudioPlayer play back BosqueAudioRegionStake objects, the track on which the region stake is placed must have a BosqueBusConfig associated with it.


A BosqueBusConfig consists of a number of input and a number of output channels and a 2-dimensional connections array that describes the "matrix" of the bus configuration (which input channel goes to which output channel at which gain). The outer array of connections has a size of numInputs, and each inner array has a size of numOutputs.


For example, imagine a track on which monophonic BosqueAudioRegionStake objects are placed. If they should be played both on the left and right speaker, a BosqueBusConfig with numInputs == 1 and numOutputs == 2 must be associated with that track, and the first element of its connections must consists of two equal gain values (e.g. [[ 1.0, 1.0 ]]).Â 


busCfgID


A unique identifier (Integer) that is used to look-up and distinguish bus configurations.



name


Returns the human readable (String) name of the config.



numInputs, numOutputs


The number of input and output channels. numInputs should correspond to the number of channels the audio regions placed on the track have. numOutputs should be <= Bosque.masterBusNumChannels.



connections


Returns (a copy of) the connections matrix of the bus configuration.
